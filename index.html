STAFF LEAVE MANAGEMENT SYSTEM

Flat Icons. An organization wants to implement an online leave absence database record for the staff. The number of leaves are different for the employees as per the designation. This system can reduce the hassle of an organization. The benefits of a well-structured database are infinite, with increased efficiency and time saving benefits. Staff Leave Management System is developed in order to make an organization hassle free. In the present day, getting a leave from the higher authority is a hassle. Application forms are filled, followed by waiting for approval, the process itself takes more than a week. This is a user-friendly application where all users regardless of age can use easily. For example: In a company leave absence database, the staff has the authority to apply for leave according to his/her convenience. The employees can also view their personal information, change password and view the number of previous holidays taken. On the other hand, the admin has more authority. The admin can manage the employees information, add new employees if necessary. The main function of the admin in this system is that to check if the employee has enough holidays or not and to approve or disapprove the leave application by specifying a proper reason for the same.

OBJECTIVES
• The main objective of this project is to maintain and manage all the details of staff and their absence details. Any staff who wants holiday can apply for holiday. The main objectives are given below:
• The project needs proper details of the type and number of holidays the staff has applied for.
• Any staff can apply and get holiday.
• All the information is maintained by the administrators based on the information available. User can update or delete his information as required.
• The approval of the holiday can automatically assigned to the employee page if the admin approved it. Else it gets disapprove.

PURPOSE
• View Staff details
• View Admin details
• Add department detail
• Apply Leaves by Staff
• Add information about the leaves by the Admin.
• Stored procedure called to view users only by the administrator.
• Trigger to auto reject when a new admin is trying to access the system.


FRONT END TOOLS: HTML, CSS, JS


BACK-END TOOL: MySQL, PHP
